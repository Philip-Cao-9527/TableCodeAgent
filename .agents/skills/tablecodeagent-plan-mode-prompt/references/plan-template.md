# TableCodeAgent Plan Mode Prompt 底稿

本文件是 `tablecodeagent-plan-mode-prompt` 的专属底稿。生成最终 prompt 时按用户任务裁剪，不保留模板说明、空占位符、旧项目路径或旧项目命令。

````text
你要在当前仓库 `d:\桌面\TableCodeAgent` 处理一次 TableCodeAgent 项目任务。必须先使用 Plan Mode，不允许跳过计划阶段直接实现。

【当前模式】

1. 本轮先进入 Plan Mode。
2. 只允许做只读探索、阅读、搜索、静态分析和必要的非破坏性核对。
3. 不要编辑文件，不要 apply patch，不要新增报告，不要删除文件，不要修改版本号，不要运行会改变仓库状态的命令。
4. 不要一开始就写死详细方案；先读真实仓库和参考材料，再提炼需要用户拍板的关键决策。

【项目背景】

1. TableCodeAgent 是面向复杂表格任务的轻量级 Coding Agent 项目，目标是把表格任务组织成可执行、可验证、可记录轨迹的程序化工作流。
2. 不要把项目包装成普通数据分析 Agent、SFT、RL、RAG、Memory 增强或 SOTA 项目。
3. 本轮只做计划和决策收敛，不直接修改 TableCodeAgent 核心运行能力。

【必读文件与参考材料】

开始提问或出计划前，必须先阅读并在进度中明确已读：

- `.codex/AGENTS.md`
- `README.md`
- `docs/reproduce/tablecodeagent_architecture.md`
- `docs/reproduce/why_table_code_agent.md`
- 本轮相关的 skill、prompt、文档或调用链文件

先运行 `rg --files .agents .codex docs` 或等价命令核对真实路径。

【关键决策点要求】

你需要从真实仓库和任务背景中提炼 2 到 4 个真正需要用户拍板的关键决策点。优先关注：

1. 目录归属和职责拆分。
2. prompt 输出边界和是否拆分为独立 skill。
3. 验证深度：只做结构校验、是否需要本地 smoke、是否涉及真实 API。
4. 版本、文档和修复报告策略。

不要把以下内容当成用户决策点：

- 能通过阅读仓库直接确认的事实。
- 函数名、变量名、琐碎文件名。
- 用户已经硬性写死的边界。
- 只有一个合理答案的工程常识。

【每个决策点的输出要求】

每个决策点必须包含：

1. 为什么必须现在决策。
2. 不同取向分别影响什么。
3. 当前从仓库事实推导出的倾向。
4. 仍需要用户确认的选择。

可以给出倾向，但不能替用户拍板。

【用户选择后的收敛要求】

用户完成关键决策后，再输出可执行计划。计划必须覆盖：

1. 文件级改动范围。
2. 调用链、模板链路或 skill 路由影响。
3. 测试与验证要求。
4. 版本、文档、报告策略。
5. 风险、回滚边界和未验证项。
6. 本轮 Plan Mode 未修改代码的明确声明。

【最终输出要求】

本轮 Plan Mode 输出只做两件事：

1. 简要说明已读材料和当前事实。
2. 提出 2 到 4 个关键决策点，等待用户选择。

不要直接实现，不要输出最终补丁，不要生成修复报告，不要把计划写成已经完成的工作。
````
